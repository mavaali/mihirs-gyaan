# Start Here

New to this? You've used an LLM as a chat box — ask a question, get an answer. This guide
takes you from there to running **skills**: reusable instruction sets that make the model do
a specific job *well*, the same way, every time.

You don't need to install anything to begin. The first win works in any chat box.

## The path

1. **[Why bother going beyond chat?](./1-why-beyond-chat.md)** — the one idea, with a
   before/after you can feel.
2. **[Your first win](./2-first-win.md)** — paste one skill into a chat box and watch the
   difference. ~10 minutes, no setup.
3. **[Go deeper](./3-go-deeper.md)** — one fork: *do you write code?* Pick your lane.
   - Coders → [set up a CLI agent](./track-a-cli-agent.md) (Claude Code, Codex, or any)
   - Knowledge workers → [set up a notes vault](./track-b-notes-vault.md) (Obsidian)
4. **[The full map](./the-full-map.md)** — every skill, by tier, with what each needs.

Want to *do* rather than read? Every page ends with a **Try this**. They're collected in
**[exercises.md](./exercises.md)** as a checklist you can work through — five small wins, each
self-verifiable, ending with you writing your own.

> [!NOTE]
> Everything here is reversible. You can't break anything by trying a skill. The whole point
> is to run one more experiment than you otherwise would.
